一、传统笔记为什么失效

每个人大概都有这样的经历：浏览器剪藏了一堆文章、Markdown 笔记存了几百篇、截图和 PDF 塞满文件夹。然后再也没有打开过。

问题不在工具，在于积累方式：

  • 记了不看：收藏即遗忘，笔记变成了心理安慰剂。
  • 碎片化严重：有感而发写一行存一个文件，半年后完全想不起来上下文。
  • 垃圾混入：随手记录的报错日志、复制粘贴的网页、拍代码的截图，占了大量空间却毫无复用价值。

笔记系统的目的不是囤积，是能被重新发现和使用。做不到这一点，再多的笔记也只是数字垃圾。

二、AI 时代的学习记录新可能

传统笔记是”单向写入”——你记，它存。AI 把这个流程变成了”双向循环”：

  1. 你输入原始材料（随手记的碎片、旧的笔记、网页摘录）。
  2. AI 帮你整理（筛选有价值的内容、合并同类项、生成结构化文本）。
  3. 你审阅并决策（确认、修改、删除、补充）。
  4. 成品发布，成为可被搜索引擎索引、可被自己或他人引用的知识。

AI 不是替代你思考，而是放大你的整理能力。你不需要从几百篇混乱笔记里手动挑拣，AI 可以做第一遍粗筛和归类。你只需要做最有价值的事：判断和决策。

利用 AI 把收藏夹吃灰的知识唤醒，让大脑从”囤积模式”切换到”重构吸收模式”。

三、LLM 具体做什么

在本博客的实践中，LLM 承担了四项核心任务：

筛选：从一批笔记中识别哪些值得保留、哪些是纯垃圾。判断标准包括内容是否有时效性、是否有复用价值、是否具有独立完整性。

合并：发现同一技术栈或同一主题下的碎片笔记，融合成一篇深度、结构化的长文。例如三篇关于 Docker 的零散记录，合并为一份完整的 Docker 实践笔记。

删除：彻底清除过期的报错日志、无意义的网页复制、纯占空间的代码截图。释放磁盘空间的同时，也让剩下的内容更纯粹。

链接：扫描文章内容，自动识别提及的已有概念，用 Obsidian 双链语法 [[概念名称]] 建立起页面之间的关联网络。

四、我的工作流

这是一个经过多次迭代沉淀下来的流程：

  1. 分批投料：从历史笔记中挑选相关笔记，控制在 5 篇之内放入待处理目录。不全丢进去的原因有两个——AI 上下文有限，以及人工审核需要可控的粒度。
  2. AI 扫描 & 提案：AI 通读本批次，输出一张表格，标注每篇的操作建议（保留、合并、删除）和目标分类。
  3. 人工审核：逐条确认或调整提案。这一步不可省略——AI 的判断需要人来把关。
  4. AI 执行：根据确认结果，执行合并、删除、写入新文章、更新索引和日志。
  5. 人工再次审核：人工检查输出质量，确保内容准确、排版规范、链接有效。
  6. 发布：Git commit → push → GitHub Actions 自动部署。

一句话总结这个工作流：人是编辑，AI 是助理。

五、踩过的坑

不要一次全丢进去。笔记数量多时，AI 的提案会变得粗糙，审核负担也成倍增加。每批控制在 5 篇以内，确保人工能逐篇检查内容的准确性和相关性。

中英双语的复杂度远超预期。语言切换、文件命名、侧栏目录过滤、文件夹命名——每一个看似简单的需求在实际实现中都可能踩坑。如果你也需要双语，建议从一开始就用最简方案，不要叠加工序。

六、分享即学习，利他即利己

互联网精神：曾无数次因为一个问题，遍历各个博客寻找一丝答案线索。那些素未谋面的作者留下的文字，帮我解决了一个又一个难题。现在轮到我留下点什么了——哪怕只帮到一个人，也是有意义的。互联网最珍贵的东西，就是这种不求回报的知识传递。

把一本书读薄：把笔记公开，意味着你必须把它写得别人看得懂。这个”让别人看懂”的过程，本身就是对知识的二次消化和深度理解。很多你以为懂了的东西，写到一半才发现其实没想清楚。写给别人看，是检验理解的最诚实的方式。

公开可能被纠正：错误藏在自己的笔记本里永远不会被发现，放出来才有机会被指正。这是最快的学习闭环——你分享，别人纠正，你改进。

灵感来源：本博客的 LLM Wiki 工作流受 Andrej Karpathy 的 gist 启发——用 LLM 作为知识库的维护者，从原始文档中增量构建结构化 wiki，人类只做策展和决策。
