## 一、上拉电阻与下拉电阻

上拉电阻（Pull-Up Resistor）和下拉电阻（Pull-Down Resistor）是数字电路中两种常见的电阻配置，用于在信号线未被驱动时确保逻辑电平稳定。

理解电压与逻辑的关系：  
逻辑 1 (High Level/高电平)：通常等同于电源电压（例如 3.3V 或 5V）。  
逻辑 0 (Low Level/低电平)：通常等同于地电压（GND，即 0V）。

### 1. 上拉电阻

- **定义**：连接在电源（Vcc）与信号线之间。当信号线未被驱动时，将其拉到高电平（逻辑 1）。
- **作用**：
  - 输入端悬空时保持确定的高电平，防止浮动状态（Undefined State）。
  - 常用于按键输入、开关等场景。
- **示例**：
  - 按键未按下 → 信号线经上拉电阻接 Vcc，因为没有其他路径，信号线处于 Vcc 电压 → 高电平（逻辑 1）。
  - 按键按下 → 信号线直接接地，由于导线的电阻远小于上拉电阻，电流直接流向地 → 低电平（逻辑 0）。

**Q：为什么需要上拉电阻？（解决”悬空”问题）**

**A：** 在数字芯片的输入引脚看来，如果它没有接到任何地方（悬空），它非常容易受到周围环境电磁干扰，导致电平在 0 和 1 之间乱跳（即”浮动”）。这种状态是逻辑灾难，因为芯片不知道自己到底读到了什么。上拉电阻的作用就是：在没有人”拉”它的时候，强制把它”拽”到高电平。

### 2. 下拉电阻

- **定义**：连接在信号线与地（GND）之间。当信号线未被驱动时，将其拉到低电平（逻辑 0）。
- **作用**：
  - 输入端悬空时保持确定的低电平。
  - 电流在无外部信号时接近于零，创建稳定的输入状态；有外部信号时电流流动，影响输入电平。
- **示例**：
  - 开关未按下 → 信号线经下拉电阻接地 → 低电平（逻辑 0）。
  - 开关按下 → 信号线经开关接 Vcc → 高电平（逻辑 1）。

### 3. 场景选择

| 协议空闲态 | 电阻类型 | 典型场景 |
|-----------|---------|---------|
| 高电平 | 上拉电阻 | UART RX、I²C（SCL/SDA） |
| 低电平 | 下拉电阻 | 部分片选信号（Chip Select） |

> **协议空闲态**：总线或接口在无数据传输时的默认电平状态。

常见协议空闲态：
- **UART**：空闲为高电平。线路保持高表示无数据，出现低电平（起始位）表示传输开始。
- **I²C**：SCL 和 SDA 空闲均为高电平。SDA 在 SCL 高电平时下降沿 = 起始条件。
- **SPI**：空闲态由时钟极性（CPOL, Clock Polarity）和片选信号决定，通常片选为高时接口空闲。

### 4. Flash 读取为什么用下拉电阻（稳定0状态）

给 Flash 芯片上电后，控制器/编程器会读取引脚状态（地址线、数据线、模式选择脚），这些引脚必须在上电瞬间稳定在规定的逻辑电平。

- **下拉电阻过大的问题**：电阻过大 → 电流过小 → 漏电流/耦合干扰/芯片内部弱上拉可能导致电平漂浮 → Flash 无法正确识别控制信号 → 读取失败。

- **类比（水流模型）**：
  - 电压源 = 水箱水位
  - 电阻 = 水管粗细
  - 电流 = 水流量
  - 下拉电阻太大 = 水管太细，水流不足以把水位拉低。

> **结论**：Flash 使用下拉电阻是为了上电瞬间确保引脚电压稳定在逻辑低，避免悬空或干扰导致错误状态，保证芯片进入正确工作模式。

### 5. 总结对比：下拉电阻 vs 上拉电阻

| **特性**   | **下拉电阻 (Pull-down)** | **上拉电阻 (Pull-up)** |
| -------- | -------------------- | ------------------ |
| **默认状态** | 逻辑 0 (GND)           | 逻辑 1 (Vcc)         |
| **主要目标** | 确保系统处于“静止/关断”状态      | 确保系统处于“准备/使能”状态    |
| **安全性**  | 防止误操作，数据更安全          | 容易在不经意间激活芯片        |
| **常见应用** | 模式选择、重置脚、写保护脚        | I2C 总线、中断信号、握手信号   |

## 三、NAND Flash数据处理注意

### 核心概念

- **数据区域（Data Area）**：存储实际内容。编程器读出的内容 = 数据 + 校验信息。
- **OOB（Out of Band，带外数据区）**：额外数据区域，用于存储 ECC（Error Correction Code，错误纠正码）。
- **实际空间分配**：需查看芯片手册，常见为 `0x800（数据）+ 0x40（校验）`。
- **运行时行为**：程序加载到内存时只加载数据内容，校验数据不会被读入。

### NOR Flash vs NAND Flash

| 类型 | 用途 | 易失性 | 典型内容 |
|------|------|--------|----------|
| SDRAM | 运行内存 | 是 | 启动后加载的代码/数据 |
| NOR Flash | 启动 + 固件（有时含完整根文件系统） | 否 | Bootloader（引导程序）、Kernel（内核）、rootfs（根文件系统） |
| NAND Flash | 存储 | 否 | rootfs、配置文件、日志 |

## 四、识别 SDRAM / NOR Flash / NAND Flash

### 核心区别速查

| 类型 | 用途 | 易失性 | 典型内容 |
|------|------|--------|----------|
| **SDRAM** | 运行内存 | ✅ 是 | 启动时加载的代码/数据 |
| **NOR Flash** | 启动 + 固件（有时含完整根文件系统） | ❌ 否 | Bootloader、Kernel、rootfs |
| **NAND Flash** | 存储 | ❌ 否 | rootfs、配置文件、日志 |

### SDRAM（同步动态随机存取存储器）

- **常见丝印标记**：`DDR`、`SDRAM`、`MT`、`H5`、`K4`

### NOR Flash

- 支持 **XIP（Execute-In-Place，片内执行）**，代码可直接在 Flash 上运行。
- **特征**：
  - 容量较小（通常 8–64 MB）
  - SPI 或并行（Parallel）接口
  - 8 脚 SOIC（SPI NOR）或 TSOP 封装
- **常见丝印标记**：`W25Q`、`MX25`、`S25`、`EN25`
- **逆向价值**：最易 dump 的芯片，偏移量干净，几乎无磨损均衡（Wear-Leveling）干扰。

### NAND Flash

- 基于块（Block）存储，需要 ECC 纠错。
- **特征**：
  - 容量较大（128 MB 以上）
  - TSOP-48 或 BGA 封装
  - 大量地址/数据线
- **常见丝印标记**：`MT29F`、`K9F`、`TC58`
- **复杂因素**：坏块（Bad Block）、ECC、磨损均衡，原始 dump 无控制器逻辑时常表现为"损坏"状态。

## 五、如何定位固件 / 文件系统位置

### Step 1：识别存储芯片
- 读取芯片丝印 → 搜索型号
- 检查封装类型

### Step 2：利用电路板布局线索
- 离 CPU 最近的 Flash → 通常是启动 NOR
- 容量最大的 Flash → 通常是 NAND（文件系统）
- SDRAM 特征：大量等长走线、附近有终端匹配电阻

### Step 3：Dump Flash
- SPI NOR → 使用 `flashrom` 工具
- NAND → 使用编程器或拆焊 + 读取器

### 文件系统布局常见模式

**纯 NOR 设备**：
```
[ bootloader ][ kernel ][ rootfs ]
```

**NOR + NAND 设备**：
```
NOR:  bootloader + kernel
NAND: rootfs + data
```

> 掌握以上布局模式，通常可直接判断应该优先 dump 哪块芯片。

### PCB 逆向技巧

- **跟踪启动引脚（Boot Pins）**：SPI 走线 → NOR；宽并行总线 → NAND；DDR 走线 → SDRAM
- **UART 启动日志**：常直接打印 `Loading kernel from SPI flash...`、`Mounting rootfs from NAND...`，即时给出答案。

### 漏洞研究方向

| 目标 | 对应存储 |
|------|---------|
| Bootloader 漏洞 | NOR Flash |
| Web/服务漏洞 | NAND rootfs |
| 运行时 RCE（远程代码执行） | 仅 SDRAM |

> **攻击面映射（Attack Surface Mapping）从存储类型开始。**

## 六、逆向分析 PCB 电路板

在不查看芯片丝印的情况下进行 PCB 逆向分析，依赖布局逻辑、走线特征、外围器件特征和信号完整性设计原则。

![](../assets/pcb-board-reverse-engineering-example.webp)

### 1. 定位 CPU / SoC（主控芯片，System on Chip）

- **核心地位**：引脚最多、封装最大、走线最密集的 BGA 芯片（球栅阵列封装）。
- **总线枢纽**：所有信号的汇聚点，大量走线从此出发通往内存、闪存、网口等。
- **供电特征**：周围环绕多组电感（方形黑色元件）和大量小型去耦电容（Decoupling Capacitor），主控需要多路低电压、大电流供电。

### 2. 定位 DRAM（内存，Dynamic Random Access Memory）

- **等长走线（蛇形线）**：判断内存最准确的特征。为保证高速数据同步，内存与 CPU 之间的走线必须长度一致，表现为弯曲并排的蛇形线。
- **紧邻 CPU**：DDR 信号频率极高，为减少干扰，内存芯片紧贴 CPU。
- **封装**：通常为长方形 BGA。

![](../assets/pcb-dram-length-matched-traces.webp)


### 3. 定位 Flash（闪存 / 固件存储）

- **走线相对稀疏**：SPI Flash 通常 8–16 引脚，走线比内存简单得多。
- **常见封装**：SOP-8 或宽体 SOP-16；大容量 NAND Flash 为长方形多引脚 TSOP。
- **位置**：靠近 CPU，但对距离要求不如内存严格。

### 4. 定位调试引脚（Debug Ports / UART）

- **排针/焊盘特征**：通常 3–4 个连续空闲焊盘或已焊接排针。
- **信号逻辑**：一个接大面积铺铜（GND），一个接 3.3V 电源，另外两个（TX/RX）直连 CPU。
- **丝印标记**：可能预留 `VCC, GND, TX, RX` 文字，或完全无标记但在 CPU 附近整齐排列。

### 5. 其他功能模块辅助判断

| 模块 | 识别特征 | 逆向逻辑 |
|------|---------|---------|
| 网络接口（PHY / Switch，物理层芯片/交换机芯片） | 靠近 RJ45 网口，连接黑色长方形网络变压器 | 变压器另一端连接的芯片 = 交换机芯片或 CPU |
| 射频（Wi-Fi / RF） | 位于屏蔽罩内，连接黑色或灰色天线馈线 | 顺天线找末端芯片 = 无线芯片（集成 SoC 则直接连主控） |
| 电源管理（PMU，Power Management Unit） | 靠近 DC 电源插座，周围有大型电感和电解电容 | 负责电压转换，为 CPU 和内存供电 |

### 逆向思路总结

1. **看走线**：蛇形线 → 内存，粗线 → 电源，密集线 → CPU。
2. **看外设**：顺网口 → PHY，顺天线 → RF，顺电源口 → 供电。
3. **看封装**：大方块 → 主控，长方块 → 内存，小方块 → Flash。

## 七、调试引脚逆向识别

### 1. Bootstrap 引脚（启动配置引脚）

不用于持续通信，但决定芯片从何处启动（Flash 启动 / USB 烧录模式）。

- **判断依据**：
  - CPU 附近一簇排列整齐的小电阻（0402 或 0201 封装），一端接 CPU 信号线，另一端接高电平或地。
  - 三个并排焊盘：中间为信号，两边分别为高电平和地，通过焊接 0Ω 电阻改变启动模式。
- **逆向逻辑**：需强制进入烧录模式时，找到这些电阻并短接。

![](../assets/pcb-bootstrap-resistors.webp)



### 2. 常见调试接口速查

| 接口 | 引脚数 | 典型排列 | 判断依据 |
|------|--------|---------|---------|
| **UART**（通用异步收发传输器） | 3–4 | 1×4 线性 | TX/RX 电压波动，电平通常为 3.3V |
| **SWD**（串行线调试，ARM 专用） | 2–4 | 1×4 或焊盘 | 仅 ARM 内核，CLK 引脚有恒定频率特征 |
| **JTAG**（联合测试行动组标准接口） | 5–20 | 2×N 阵列或连续焊盘 | 引脚间有匹配电阻，信号直通 CPU 核心 |
| **Boot Pin**（启动配置引脚） | 1–4 | 电阻群 | 位于 CPU 旁，通过电阻连接 VCC 或 GND |

## 八、非标准化接口的跳线点查找

如果排除了 UART、SWD、JTAG 等标准调试接口，需要从非标准化调试或底层硬件配置入手。

### 1. 寻找启动配置引脚（Bootstrap / Strapping Pins）

- 寻找 CPU 附近成排的 0Ω 电阻或空焊盘电阻位（一端接信号，另一端接 3.3V 或 GND）。
- 改变其中一个引脚的电平（如高→低），可能使系统进入救援模式（Recovery Mode），这本身就是一种调试手段。

### 2. 利用过孔矩阵（Via Matrix）进行信号探测

- BGA 芯片周围的过孔是天然测试点。
- **操作**：用万用表通断档（Continuity）——表笔一头按在 Flash 芯片引脚（如 CLK 时钟线），另一头在 CPU 附近过孔阵列中扫过，找出直通核心总线的过孔。
- **刮开绿油（Solder Mask）**：对未露出焊盘的过孔，用手术刀轻刮表面绿色阻焊漆，露出铜层后可焊接飞线（Jump Wire）进行逻辑分析。

### 3. 寻找 I²C / SPI 嗅探点

- **定位对象**：板子上 8 脚或 16 脚小芯片（如 EEPROM 或传感器）。
- **I²C**：寻找有一对上拉电阻（通常 4.7kΩ）连接到 3.3V 的线路。
- **SPI**：寻找连接到 Winbond Flash 芯片的 1、2、5、6 脚。
- 这些总线上常承载关键配置信息或加密密钥，挂载逻辑分析仪可实现旁路调试。

### 4. 识别孤立测试点（Test Point, TP）

板上散布的金色小圆点，通常带有 `TP` 编号：

- **静态 TP**：测量电压（1.1V 核心电压、1.8V 内存电压、3.3V I/O 电压）。
- **动态 TP**：关键时钟信号（CLK）或复位信号（RESET）。
- **判断**：用示波器测量，恒定 25MHz/40MHz 正弦波 = 晶振时钟测试点；上电瞬间跳变后保持高电平 = Reset 引脚。

### 5. 盲绘电路图

1. 确定地平面（GND）：大面积覆铜或电解电容负极。
2. 电源树溯源：从 DC 电源口 → 顺电感（标 `1R2` / `4R7` 的方块）→ 电感后过孔 = 主电源导轨。
3. 互连映射：
   - CPU → 内存：等长蛇形走线组。
   - CPU → 网口：顺网络变压器引脚反推。
   - CPU → 存储：顺 Flash 芯片 8 脚反推。

### 总结：最值得跳线的位置

> **CPU 与 Flash 芯片之间的 6 根 SPI 总线走线。**

接上飞线后，可在系统运行时读取所有底层指令流，甚至挂载外部 Flash 替换原厂固件。
